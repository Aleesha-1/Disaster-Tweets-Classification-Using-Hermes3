# Disaster Tweets Classification Using Hermes 3

## Overview

This project presents an AI-powered disaster tweet classification system designed to identify whether a tweet is related to a real disaster or not. The system combines traditional Machine Learning techniques with Large Language Model (LLM) insights generated by Hermes 3 to improve tweet classification performance.

The primary objective of this project is to filter noisy social media content and detect genuine emergency-related information from Twitter data.

## Dataset

The project uses a dataset containing disaster-related and non-disaster-related tweets.

**Dataset Shape:** `7613 rows × 5 columns`

After preprocessing, only the following columns were retained:

* `text`
* `target`

Where:

* `text` → Tweet content
* `target` → Label (0 = Non-Disaster, 1 = Disaster)

## Project Workflow

### 1. Data Exploration

The dataset was explored using:

* `info()`
* `describe()`
* `shape`
* `head()`
* `tail()`
* `isnull().sum()`
* `value_counts()`

### 2. Text Preprocessing

A custom `clean_text()` function was developed to preprocess tweet text.

The preprocessing pipeline included:

* Converting text to lowercase
* Removing URLs
* Removing punctuation
* Removing numerical values
* Cleaning noisy text

The cleaned tweets were stored in a new column:

clean_text

### 3. LLM-Based Feature Engineering

Hermes 3 was integrated to generate an additional feature called:

LLM_Flag

A custom function:

get_llm_flag(tweet)

was used to analyze each cleaned tweet and assign an LLM-generated flag.

To avoid losing progress during long processing, checkpointing was implemented:

* Progress saved after every 100 tweets.
* Intermediate results stored in `progress.csv`.
* Processing automatically resumes if previous progress exists.

Final processed data is stored in:

final_results.csv

### 4. Feature Extraction

Tweet text was transformed into numerical representations using TF-IDF Vectorization.

The TF-IDF features were then combined with the LLM-generated feature:

X = np.hstack((X_text.toarray(), df[['LLM_Flag']].values))

### 5. Model Training

The dataset was split into training and testing sets using an 80:20 ratio.

Two Machine Learning models were trained:

#### Bernoulli Naive Bayes

Parameters:

alpha = 0.5

#### Decision Tree Classifier

Parameters:

criterion='entropy'
max_depth=5
min_samples_split=10
random_state=42

## Model Evaluation

### Bernoulli Naive Bayes

* Accuracy: **81%**
* F1 Score: **0.76**

### Decision Tree

* Accuracy: **75%**
* F1 Score: **0.63**

The Bernoulli Naive Bayes model outperformed the Decision Tree model.

## Prediction System

A custom prediction function:

predict_with_both_models()

was developed to:

1. Clean incoming text.
2. Generate LLM flag using Hermes 3.
3. Convert text into TF-IDF vectors.
4. Combine TF-IDF features with LLM flag.
5. Predict using both models.

Example tweet:

> "The song is the bomb"

Both models correctly classified this tweet as:

Non-Disaster

## Results

The project demonstrates that combining traditional Machine Learning techniques with LLM-generated features can improve disaster tweet classification performance.

Among the evaluated models, Bernoulli Naive Bayes achieved the best performance.
